<span data-ttu-id="a24d4-104">Quando escreve [Configurações](configurations.md), você adiciona [Blocos de recurso](../resources/resources.md) para configurar aspectos de um nó de destino.</span><span class="sxs-lookup"><span data-stu-id="a24d4-104">When you write [Configurations](configurations.md), you add [Resource blocks](../resources/resources.md) to configure aspects of a target Node.</span></span> <span data-ttu-id="a24d4-105">Conforme você continua adicionando blocos de recurso, suas configurações podem se tornar muito grandes e difíceis de gerenciar.</span><span class="sxs-lookup"><span data-stu-id="a24d4-105">As you continue to add Resource blocks, your Configurations can grow quite large and cumbersome to manage.</span></span> <span data-ttu-id="a24d4-106">Um dos desafios é a ordem aplicada dos blocos de recurso.</span><span class="sxs-lookup"><span data-stu-id="a24d4-106">One such challenge is the applied order of your resource blocks.</span></span> <span data-ttu-id="a24d4-107">Normalmente, os recursos são aplicados na ordem em que são definidos dentro da configuração.</span><span class="sxs-lookup"><span data-stu-id="a24d4-107">Typically resources are applied in the order they are defined within the Configuration.</span></span> <span data-ttu-id="a24d4-108">Conforme sua configuração se torna maior e mais complexa, você pode usar a chave `DependsOn` para alterar a ordem aplicada de seus recursos especificando que um recurso depende de outro.</span><span class="sxs-lookup"><span data-stu-id="a24d4-108">As your Configuration grows larger and more complex, you can use the `DependsOn` key to change the applied order of your resources by specifying that a resource depends on another resource.</span></span>

<span data-ttu-id="a24d4-109">A chave `DependsOn` pode ser usada em qualquer bloco de recurso.</span><span class="sxs-lookup"><span data-stu-id="a24d4-109">The `DependsOn` key can be used in any Resource block.</span></span> <span data-ttu-id="a24d4-110">Ela é definida com o mesmo mecanismo de chave/valor que outras chaves de recurso.</span><span class="sxs-lookup"><span data-stu-id="a24d4-110">It is defined with the same key/value mechanism as other Resource keys.</span></span> <span data-ttu-id="a24d4-111">A chave `DependsOn` espera uma matriz de cadeias de caracteres com a sintaxe a seguir.</span><span class="sxs-lookup"><span data-stu-id="a24d4-111">The `DependsOn` key expects an array of strings with the following syntax.</span></span>

```
DependsOn = '[<Resource Type>]<Resource Name>', '[<Resource Type>]<Resource Name'
```

<span data-ttu-id="a24d4-112">O exemplo a seguir configura uma regra de firewall após habilitar e configurar o perfil público.</span><span class="sxs-lookup"><span data-stu-id="a24d4-112">The following example configures a firewall rule after enabling and configuring the public profile.</span></span>

```powershell
# Install the NetworkingDSC module to configure firewall rules and profiles.
Install-Module -Name NetworkingDSC

Configuration ConfigureFirewall
{
    Import-DSCResource -Name Firewall, FirewallProfile
    Node localhost
    {
        Firewall Firewall
        {
            Name                  = 'IIS-WebServerRole-HTTP-In-TCP'
            Ensure                = 'Present'
            Enabled               = 'True'
            DependsOn             = '[FirewallProfile]FirewallProfilePublic'
        }

        FirewallProfile FirewallProfilePublic
        {
            Name = 'Public'
            Enabled = 'True'
            DefaultInboundAction = 'Block'
            DefaultOutboundAction = 'Allow'
            AllowInboundRules = 'True'
            AllowLocalFirewallRules = 'False'
            AllowLocalIPsecRules = 'False'
            NotifyOnListen = 'True'
            LogFileName = '%systemroot%\system32\LogFiles\Firewall\pfirewall.log'
            LogMaxSizeKilobytes = 16384
            LogAllowed = 'False'
            LogBlocked = 'True'
            LogIgnored = 'NotConfigured'
        }
    }
}

ConfigureFirewall -OutputPath C:\Temp\
```
